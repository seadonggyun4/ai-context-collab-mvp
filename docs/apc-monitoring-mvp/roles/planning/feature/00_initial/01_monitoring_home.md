## 목적

모니터링 홈은 APC(농산물산지유통센터) 데이터의 오늘 상태를 가장 먼저 판단하는 화면이다.

사용자는 이 화면에서 상세 로그를 읽기 전에 다음 질문에 답할 수 있어야 한다.

- 오늘 데이터가 믿을 수 있는 상태인가?
- 어느 APC가 지연, 오류, 미수신 상태인가?
- 품목과 입고/선별 기준으로 문제가 어디에 집중되어 있는가?
- 지금 바로 확인해야 할 이슈가 있는가?

## 화면 구성

### KPI 카드

상단에는 전체 상태를 요약하는 KPI 카드를 배치한다.

| 카드 | 의미 |
| --- | --- |
| 전체 APC 수 | 모니터링 기준에 포함된 APC 수 |
| 정상 APC 수 | 기준 기간 안에 정상 수신/정제/조회 가능한 APC 수 |
| 지연 APC 수 | 기대 수신 시각을 초과한 APC 수 |
| 오류 APC 수 | 검증, 저장, 정제, 적재, 조회 중 오류가 있는 APC 수 |
| 미수신 APC 수 | 기준 기간 내 수신 이력이 없는 APC 수 |

KPI 카드는 단순 숫자 표시가 아니라 클릭 가능한 요약 진입점으로 사용한다. 예를 들어 `오류 APC 수`를 선택하면 오류 상태 APC만 matrix와 이슈 목록에 필터링된다.

### APC별 상태 matrix

중앙에는 APC, 품목, 입고/선별을 교차해서 상태를 보여주는 matrix를 둔다.

| APC | 감귤 입고 | 감귤 선별 | 당근 입고 | 당근 선별 |
| --- | --- | --- | --- | --- |
| 남원 | 상태 | 상태 | 상태 | 상태 |
| 위미 | 상태 | 상태 | 상태 | 상태 |
| 서귀 | 상태 | 상태 | 상태 | 상태 |
| 중문 | 상태 | 상태 | 상태 | 상태 |
| 구좌 | 상태 | 상태 | 상태 | 상태 |

상태는 `정상`, `지연`, `오류`, `미수신`, `기준 미정`으로 표시한다. 색상만 사용하지 않고 텍스트 label을 함께 노출한다.

### 최근 이슈 목록

하단에는 최근 발생한 이슈를 목록으로 표시한다.

| 컬럼 | 설명 |
| --- | --- |
| 발생 시각 | 이슈가 처음 또는 최근 확인된 시각 |
| APC | 이슈가 발생한 APC |
| 품목 | 감귤, 당근 등 |
| 구분 | 입고 또는 선별 |
| 상태 | 지연, 오류, 미수신 등 |
| 조치 필요 여부 | 운영자 확인이 필요한지 여부 |

최근 이슈는 조치가 필요한 항목을 상단에 우선 정렬한다.

## UX 정책

- 기본 조회 기준은 당일이다.
- 사용자가 별도 필터를 적용하지 않아도 오늘의 상태가 즉시 보여야 한다.
- 오류와 미수신은 정상 상태보다 항상 높은 우선순위로 노출한다.
- matrix cell을 선택하면 해당 조건이 적용된 `수신 현황` 또는 `데이터 품질 이슈` 상세로 이동한다.
- 최근 이슈 row를 선택하면 이슈 상세와 운영 조치 이력으로 이동한다.
- 마지막 갱신 시각을 화면 상단에 표시한다.

## MVP 범위

- matrix는 남원, 위미, 서귀, 중문, 구좌를 기본 행으로 정의한다.
- 품목은 감귤과 당근을 기준으로 한다.
- 중문은 입고/선별 분리 기준으로 표시한다.
- 위미는 정제 미구현 또는 제한 상태를 별도 표시할 수 있어야 한다.
- 외부 알림 발송은 제외하고 화면 내 경고만 제공한다.
