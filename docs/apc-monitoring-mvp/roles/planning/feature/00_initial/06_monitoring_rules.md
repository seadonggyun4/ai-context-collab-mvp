## 목적

모니터링 기준 설정은 APC별 수신 주기, 지연 허용 시간, 품질 기준, 심각도 기준을 관리하는 화면이다.

기준이 명확하지 않으면 `지연`, `미수신`, `오류` 판단이 사람마다 달라진다. 이 화면은 운영 판단 기준을 문서와 화면에서 동일하게 유지하기 위한 관리 기능이다.

## 설정 항목

| 항목 | 설명 |
| --- | --- |
| APC별 기대 수신 주기 | APC마다 데이터가 들어와야 하는 기준 주기 |
| 입고/선별별 허용 지연 시간 | 입고와 선별 데이터별 지연 허용 범위 |
| 품목별 필수값 기준 | 감귤, 당근 등 품목별 필수 필드 |
| 중복 판단 기준 | 동일 데이터로 볼 조건 |
| 오류 심각도 기준 | 높음, 보통, 낮음 판단 기준 |
| 알림 대상/알림 방식 | 후속 범위의 알림 대상과 방식 |

## 권한 정책

- 관리자만 기준을 수정할 수 있다.
- 일반 운영자는 기준을 조회할 수 있지만 변경할 수 없다.
- 기준 변경은 변경 전/후 값과 변경 사유를 이력으로 남긴다.

## 화면 구성

### 기준 목록

기준 목록은 APC, 품목, 입고/선별 단위로 표시한다.

| 컬럼 | 설명 |
| --- | --- |
| APC | 기준이 적용되는 APC |
| 품목 | 감귤, 당근 등 |
| 구분 | 입고 또는 선별 |
| 기대 수신 주기 | 정상 수신 판단 기준 |
| 허용 지연 시간 | 지연 상태 판단 기준 |
| 필수값 기준 | 품질 검사 기준 요약 |
| 최종 변경자 | 마지막으로 기준을 수정한 사용자 |
| 최종 변경 시각 | 마지막 수정 시각 |

### 기준 변경 화면

기준 변경 시에는 변경 전/후 값을 나란히 보여준다.

- 변경 전 값
- 변경 후 값
- 변경 사유
- 적용 시작 시점
- 영향 받는 APC/품목/구분

## UX 정책

- 기준 변경은 저장 전 확인 절차를 둔다.
- 변경 전/후 차이를 명확히 표시한다.
- 기준이 비어 있는 항목은 `기준 미정`으로 표시한다.
- 기준 미정 항목은 모니터링 홈과 수신 현황에서 별도로 표시한다.
- 알림 발송 설정은 MVP에서는 비활성 또는 후속 범위로 표시한다.

## MVP 기본값

- 중문은 입고와 선별을 분리해서 관리한다.
- 위미는 `정제 미구현/제한` 상태를 별도 표시한다.
- 상세 알림 발송은 후속 범위로 두고, MVP에서는 화면 내 경고만 정의한다.
- 외부 알림 채널 설정은 문서에만 남기고 실제 화면 우선순위에서는 낮춘다.
