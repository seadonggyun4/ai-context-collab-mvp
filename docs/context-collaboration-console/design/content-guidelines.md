# Content Guidelines

## Tone

- 짧고 구체적이며 검증 가능한 표현을 사용한다.
- 시스템의 확신과 한계를 함께 보여준다.
- 사용자가 기술 용어를 몰라도 결과를 이해할 수 있게 하고, 원본 경로는 보조 정보로 둔다.

## 용어

| 사용 | 사용하지 않음 |
| --- | --- |
| AI 제안 | AI의 결정 |
| 영향 분석 | 지능형 인사이트 |
| 활성 Context | 최신 AI 지식 |
| 검증 필요 | 거의 완료 |
| 실행하지 않음 | 문제 없음 |
| 변경 승인 | 마법처럼 적용 |

## 상태 문구

- `승인 대기 · 기획과 QA 영향 검토가 필요합니다.`
- `검증 실패 · 2개 테스트가 통과하지 않아 완료할 수 없습니다.`
- `수동 확인 필요 · 모바일 레이아웃은 아직 확인되지 않았습니다.`
- `Context 변경됨 · 검토 이후 기준 문서가 갱신되었습니다.`

## 표면 금지어

`MVP`, `프로토타입`, `데모`, `AI-powered`, `Next-generation`, `Revolutionary`, `차세대`, `혁신적인`, `사내 서비스 모델`.

해당 단어는 내부 개발 문서에서 범위나 제작 단계를 설명할 때만 허용하며, 사용자 UI fixture와 production copy에는 포함하지 않는다.
