# Planning Context

## 책임

- 변경 요청의 사용자 의도와 검증 가능한 수용 기준을 정의한다.
- 화면 목록보다 사용자의 판단·행동·예외 흐름을 먼저 정의한다.
- AI 제안과 사람의 확정 내용을 분리한다.
- 각 요구사항을 `REQ-*`, 시나리오를 `FLOW-*` ID로 추적한다.

## 완료 게이트

- 정상·빈 상태·로딩·오류·권한 없음·반려·검증 실패가 정의돼야 한다.
- 각 핵심 행동의 선행 조건과 성공·실패 결과가 있어야 한다.
- 화면 문구에 제작 단계나 과장된 AI 표현이 없어야 한다.
