# Planning Standards

## 목적

기획 문서는 구현할 화면과 기능을 설명하는 문서가 아니라, 프로젝트 참여자가 같은 문제를 같은 기준으로 이해하게 만드는 기준 문서다.

## 작성 원칙

- Project Context의 목적, 사용자, 용어 정의를 먼저 따른다.
- 기능 정의는 사용자 행동, 시스템 반응, 예외 상황을 함께 작성한다.
- 화면 정책은 UI 형태보다 업무 판단 기준을 우선 작성한다.
- 상위 문서에 이미 있는 설명은 반복하지 않고 링크로 참조한다.
- 변경 사항은 반드시 영향 범위를 함께 기록한다.

## 필수 항목

- 기능 목적
- 사용자 및 권한
- 업무 흐름
- 화면 흐름
- 입력/조회/다운로드 정책
- 예외 처리
- 완료 기준
- QA 관점 확인 항목
