# Impact Analysis Guide

## 목적

impact-analysis 문서는 기획, 퍼블리싱, 개발, QA 중 하나의 기준이 변경되었을 때 다른 직군 문서와 구현 범위에 어떤 영향이 있는지 기록하는 문서다.

AI는 변경 요청을 받을 때 이 문서를 기준으로 영향 범위를 정리하고, 필요한 문서 갱신 대상을 제안한다.

## 작성 시점

- Project Context의 목적, 사용자, 요구사항, 제약사항이 변경될 때
- Planning 메뉴/기능/상태/예외 정책이 변경될 때
- Publishing 디자인 시스템, 토큰, 컴포넌트 정책이 변경될 때
- Development API, 타입, 데이터 흐름, 배포 방식이 변경될 때
- QA 체크 항목 또는 수용 기준이 변경될 때

## 영향 분석 항목

| 항목 | 설명 |
| --- | --- |
| 변경 요약 | 무엇이 변경되었는가 |
| 변경 원인 | 왜 변경되었는가 |
| 영향 문서 | 수정이 필요한 문서 목록 |
| 기획 영향 | 사용자 흐름, 메뉴, 상태, 예외 정책 변경 여부 |
| 퍼블리싱 영향 | 컴포넌트, 토큰, 반응형, 접근성 변경 여부 |
| 개발 영향 | API, 타입, fixture, adapter, 배포 변경 여부 |
| QA 영향 | 테스트 케이스와 체크표 변경 여부 |
| 리스크 | 누락 시 발생할 수 있는 문제 |
| 후속 조치 | 문서 또는 구현 변경 TASK |

## 파일명 규칙
