# Development Standards

## 목적

개발 문서는 구현자가 같은 기술 기준과 검증 기준으로 기능을 추가할 수 있도록 API, 데이터, 배치, 테스트 정책을 정리한다.

## 작성 원칙

- 기존 JADX 서비스의 도메인 구조와 네이밍을 따른다.
- API 변경은 요청/응답 스키마, 권한, 오류 코드를 함께 정의한다.
- 배치 변경은 입력, 출력, 실행 주기, 재처리 정책을 함께 정의한다.
- 데이터 품질 검증은 수집 시점, 정제 시점, 적재 시점 중 어디에서 수행되는지 명확히 한다.
- 테스트는 단위 테스트, API 테스트, 배치 테스트, 화면 연동 테스트를 분리한다.

## 필수 항목

- 기술 스택
- API 계약
- 데이터 모델
- 배치/DAG 정책
- 오류 처리
- 로깅/모니터링
- 테스트 기준
