# Publishing Standards

## 목적

퍼블리싱 문서는 화면 구현자가 동일한 UI 기준으로 빠르게 판단할 수 있도록 디자인 시스템, 컴포넌트, 반응형, 접근성 기준을 정리한다.

## 작성 원칙

- 기존 JADX 프론트엔드의 UI 라이브러리와 컴포넌트 패턴을 우선 사용한다.
- 데이터 조회 화면은 대시보드보다 업무 도구로 보이게 구성한다.
- 필터, 테이블, 상태 배지, 차트, 다운로드 버튼의 상태를 명확히 구분한다.
- 로딩, 빈 데이터, 오류, 권한 없음 상태를 반드시 정의한다.
- 데이터 품질 상태는 색상만으로 구분하지 않고 텍스트와 아이콘을 함께 사용한다.

## 필수 항목

- 화면 레이아웃
- 컴포넌트 정책
- 상태 표시 정책
- 테이블/차트 정책
- 반응형 기준
- 접근성 기준
