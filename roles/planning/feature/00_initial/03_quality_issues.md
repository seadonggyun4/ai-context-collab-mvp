## 목적

데이터 품질 이슈 화면은 수신된 APC 데이터가 조회와 통계에 사용해도 되는 상태인지 판단하는 화면이다.

이 화면은 단순 오류 목록이 아니라, 사용자가 `어떤 데이터가`, `왜 위험한지`, `어떤 조치가 필요한지`를 이해하게 하는 기획 화면이다.

## 이슈 유형

| 유형 | 설명 |
| --- | --- |
| 필수값 누락 | APC명, 품목, 입고/선별 구분, 수량, 중량 등 필수값이 없음 |
| 형식 오류 | 날짜, 숫자, 코드 형식이 기대 규칙과 다름 |
| 중복 의심 | 동일 APC/품목/구분/시각 기준으로 중복 가능성이 있음 |
| 비정상 중량/수량 | 중량 또는 수량이 비정상적으로 작거나 큰 값 |
| 미지원 APC/품목 | 현재 기준에 없는 APC 또는 품목이 들어옴 |
| 정제 실패 | origin 저장은 되었으나 refined 저장 또는 변환에 실패 |

## 상세 화면

이슈 row를 선택하면 상세 패널 또는 drawer를 표시한다.

상세 화면에는 다음 정보를 제공한다.

- 오류 요약
- 영향 데이터 범위
- 샘플 row
- 통계/엑셀 다운로드 영향 여부
- 사용자가 해야 할 조치
- 관련 수신 현황 또는 파이프라인 단계 이동
- 운영 조치 이력

샘플 row는 문제 판단에 필요한 최소 필드만 보여준다. 전체 원천 데이터는 권한이 있는 사용자에게만 제공한다.

## 이슈 상태

| 상태 | 의미 |
| --- | --- |
| 미확인 | 아직 운영자가 확인하지 않은 이슈 |
| 확인중 | 운영자가 원인을 확인 중인 이슈 |
| 조치완료 | 조치가 끝났고 후속 확인이 완료된 이슈 |
| 무시 | 데이터 영향이 없거나 의도된 값으로 판단한 이슈 |

## UX 정책

- 미확인이며 심각도가 높은 이슈를 상단에 우선 노출한다.
- 오류 유형은 개발자 로그명이 아니라 운영자가 이해할 수 있는 이름으로 표시한다.
- 엑셀 다운로드 전 품질 이슈가 있으면 경고를 표시한다.
- 시각화 화면에 영향을 주는 이슈가 있으면 차트 상단에 데이터 신뢰도 경고를 표시한다.
- `무시` 처리된 이슈도 삭제하지 않고 이력에 남긴다.
