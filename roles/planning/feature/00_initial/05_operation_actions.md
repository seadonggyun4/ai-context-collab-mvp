## 목적

운영 조치 내역은 이슈를 단순히 표시하는 데서 끝내지 않고, 누가 언제 무엇을 확인했고 어떤 조치를 했는지 남기는 화면이다.

APC 모니터링은 오류 탐지만으로는 충분하지 않다. 같은 이슈가 반복되는지, 조치가 완료되었는지, 담당자가 상황을 인지했는지를 추적할 수 있어야 한다.

## 주요 기능

| 기능 | 설명 |
| --- | --- |
| 이슈 상태 변경 | 미확인, 확인중, 조치완료, 무시 상태로 변경 |
| 담당자 메모 | 운영자가 확인한 내용과 다음 행동을 기록 |
| 조치 완료 기록 | 완료 시각, 조치자, 조치 내용을 기록 |
| 재발 여부 확인 | 동일 조건의 이슈가 다시 발생했는지 확인 |
| 관련 화면 이동 | 품질 이슈, 수신 현황, 파이프라인 추적으로 이동 |

## 조치 이력 timeline

이슈 상세 하단에는 조치 이력을 timeline으로 표시한다.

| 항목 | 설명 |
| --- | --- |
| 시각 | 조치가 기록된 시각 |
| 작성자 | 조치 또는 메모를 남긴 사용자 |
| 상태 변경 | 변경 전/후 상태 |
| 조치 내용 | 운영자가 이해할 수 있는 문장형 기록 |
| 관련 링크 | 필요 시 파이프라인 추적 또는 품질 이슈 상세 |

## UX 정책

- 조치 기록은 개발자용 로그가 아니라 운영자용 문장으로 남긴다.
- 조치 완료 후에도 원본 이슈는 삭제하지 않는다.
- `무시` 상태로 처리한 경우에도 사유 입력을 요구한다.
- 상태 변경과 메모 작성은 같은 화면에서 처리할 수 있어야 한다.
- 반복 발생 이슈는 이전 조치 이력을 함께 볼 수 있어야 한다.

## 화면 행동

1. 사용자가 품질 이슈 또는 최근 이슈 목록에서 항목을 선택한다.
2. 상세 화면 하단의 조치 이력 timeline을 확인한다.
3. 상태를 `확인중`으로 변경하고 메모를 남긴다.
4. 원인 확인 후 `조치완료` 또는 `무시`로 변경한다.
5. 동일 이슈가 재발하면 이전 조치 이력을 함께 표시한다.

## MVP 범위

- 조치 이력은 화면 정책과 항목 정의까지만 포함한다.
- 외부 협업 도구 연동, 자동 ticket 생성, 알림 발송은 후속 범위다.
- 이력 삭제 기능은 제공하지 않는다.
